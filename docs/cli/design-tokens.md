# Design Tokens

Design tokens are the foundational variables that define your authentication pages' visual identity. They capture colors, spacing, border radii, and typography in a single YAML file that compiles to CSS custom properties.

## What Are Design Tokens?

Design tokens are named values that represent design decisions. Instead of hard-coding `#4F46E5` throughout your templates, you define it once as a token (`colors.primary`) and reference it everywhere as `var(--af-color-primary)`. This makes it easy to update your entire design system by changing a single value.

## Token File

Tokens are defined in `tokens/design-tokens.yaml` in your project directory:

```yaml
colors:
  primary: "#4F46E5"
  secondary: "#7C3AED"
  background: "#FFFFFF"
  text: "#1F2937"
  error: "#DC2626"
  success: "#16A34A"
  warning: "#F59E0B"
  info: "#3B82F6"

spacing:
  xs: "4px"
  sm: "8px"
  md: "16px"
  lg: "24px"
  xl: "32px"
  2xl: "48px"

radius:
  sm: "4px"
  md: "8px"
  lg: "16px"
  full: "9999px"

typography:
  font_family: "'Inter', system-ui, -apple-system, sans-serif"
  font_size_xs: "0.75rem"
  font_size_sm: "0.875rem"
  font_size_md: "1rem"
  font_size_lg: "1.25rem"
  font_size_xl: "1.5rem"
  font_size_2xl: "2rem"
  font_weight_normal: "400"
  font_weight_medium: "500"
  font_weight_bold: "700"
  line_height_tight: "1.25"
  line_height_normal: "1.5"
  line_height_relaxed: "1.75"
```

## Configuration

The `cpi-auth.config.yaml` tokens section controls how tokens are built:

```yaml
tokens:
  directory: ./tokens
  output: ./styles/tokens.css
```

| Key | Description |
|-----|-------------|
| `directory` | Where token YAML files are located |
| `output` | Where the compiled CSS file is written |

## Token Categories

### Colors

Define the color palette for your authentication pages.

| Token | CSS Property | Default | Usage |
|-------|-------------|---------|-------|
| `colors.primary` | `--af-color-primary` | `#4F46E5` | Buttons, links, active states |
| `colors.secondary` | `--af-color-secondary` | `#7C3AED` | Secondary actions, accents |
| `colors.background` | `--af-color-background` | `#FFFFFF` | Page and card backgrounds |
| `colors.text` | `--af-color-text` | `#1F2937` | Body text, headings, labels |
| `colors.error` | `--af-color-error` | `#DC2626` | Error messages, invalid fields |
| `colors.success` | `--af-color-success` | `#16A34A` | Success messages, valid fields |
| `colors.warning` | `--af-color-warning` | `#F59E0B` | Warning messages |
| `colors.info` | `--af-color-info` | `#3B82F6` | Informational messages |

### Spacing

Consistent spacing scale for margins, padding, and gaps.

| Token | CSS Property | Default |
|-------|-------------|---------|
| `spacing.xs` | `--af-spacing-xs` | `4px` |
| `spacing.sm` | `--af-spacing-sm` | `8px` |
| `spacing.md` | `--af-spacing-md` | `16px` |
| `spacing.lg` | `--af-spacing-lg` | `24px` |
| `spacing.xl` | `--af-spacing-xl` | `32px` |
| `spacing.2xl` | `--af-spacing-2xl` | `48px` |

### Border Radius

Corner rounding for buttons, inputs, cards, and containers.

| Token | CSS Property | Default |
|-------|-------------|---------|
| `radius.sm` | `--af-radius-sm` | `4px` |
| `radius.md` | `--af-radius-md` | `8px` |
| `radius.lg` | `--af-radius-lg` | `16px` |
| `radius.full` | `--af-radius-full` | `9999px` |

### Typography

Font settings for text rendering.

| Token | CSS Property | Default |
|-------|-------------|---------|
| `typography.font_family` | `--af-font-family` | `'Inter', sans-serif` |
| `typography.font_size_sm` | `--af-font-size-sm` | `0.875rem` |
| `typography.font_size_md` | `--af-font-size-md` | `1rem` |
| `typography.font_size_lg` | `--af-font-size-lg` | `1.25rem` |
| `typography.font_weight_normal` | `--af-font-weight-normal` | `400` |
| `typography.font_weight_medium` | `--af-font-weight-medium` | `500` |
| `typography.font_weight_bold` | `--af-font-weight-bold` | `700` |

## Generated CSS

```bash
cpi-auth tokens build -o ./styles/tokens.css
```

## Using Tokens in Templates

Reference tokens in your template CSS using `var()`:

```css
body {
  font-family: var(--af-font-family);
  background-color: var(--af-color-background);
  color: var(--af-color-text);
  line-height: var(--af-line-height-normal);
}

.form-container {
  padding: var(--af-spacing-xl);
  max-width: 420px;
  margin: 0 auto;
}

.form-field {
  margin-bottom: var(--af-spacing-md);
}

.form-field input {
  width: 100%;
  padding: var(--af-spacing-sm) var(--af-spacing-md);
  border: 1px solid #d1d5db;
  border-radius: var(--af-radius-md);
  font-size: var(--af-font-size-md);
  font-family: var(--af-font-family);
}

.form-field input:focus {
  outline: 2px solid var(--af-color-primary);
  outline-offset: -1px;
}

.submit-button {
  width: 100%;
  padding: var(--af-spacing-sm) var(--af-spacing-lg);
  background-color: var(--af-color-primary);
  color: white;
  border: none;
  border-radius: var(--af-radius-md);
  font-size: var(--af-font-size-md);
  font-weight: var(--af-font-weight-medium);
  cursor: pointer;
}

.error-message {
  color: var(--af-color-error);
  font-size: var(--af-font-size-sm);
  margin-top: var(--af-spacing-xs);
}

.success-message {
  color: var(--af-color-success);
  padding: var(--af-spacing-md);
  background-color: #f0fdf4;
  border-radius: var(--af-radius-md);
}
```

## WCAG Contrast Validation

The CLI validates color token combinations against WCAG 2.1 contrast requirements.

```bash
cpi-auth tokens validate
```

### Checked Combinations

| Foreground | Background | Required Ratio | Level |
|------------|-----------|---------------|-------|
| `colors.primary` | `colors.background` | 4.5:1 | AA (normal text) |
| `colors.text` | `colors.background` | 4.5:1 | AA (normal text) |
| `colors.error` | `colors.background` | 4.5:1 | AA (normal text) |
| White (#FFFFFF) | `colors.primary` | 4.5:1 | AA (button text) |

A warning is issued if any combination fails WCAG AA. Failing AAA is noted but not treated as an error.

## Example Workflow

### 1. Initialize the project

```bash
cpi-auth init --server http://localhost:5054
cpi-auth login -e admin@example.com -p password
cpi-auth pull
```

### 2. Edit tokens

Open `tokens/design-tokens.yaml` and change the color palette:

```yaml
colors:
  primary: "#2563EB"     # Changed from indigo to blue
  secondary: "#7C3AED"
  background: "#F9FAFB"  # Slightly off-white
  text: "#111827"         # Darker text
```

### 3. Build and validate

```bash
cpi-auth tokens build
cpi-auth tokens validate
```

### 4. Preview locally

```bash
cpi-auth dev --port 4400
# Open http://localhost:4400 to see changes
```

### 5. Review diff and push

```bash
cpi-auth diff
cpi-auth push --dry-run
cpi-auth push
```
